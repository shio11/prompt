# Copilot Studio「仕様書 変化点抽出 → Excel出力」たたき台

参考: [Copilot Studio で AI プロンプトを介してコードインタプリターを利用する（Qiita / Takashi_Masumori）](https://qiita.com/Takashi_Masumori/items/0c5ee585d02575764c8e)

## 1. 全体構成

```
[ユーザー発話]
  「対象の仕様書から変化点を抽出して」
        │ (トリガーフレーズ / 生成AIマッチ)
        ▼
[トピック: 仕様書変化点抽出]
  ① 旧版仕様書アップロード受付
  ② 新版仕様書アップロード受付
  ③ AIプロンプト(ツール)呼び出し
        │ 入力: 旧版ファイル, 新版ファイル
        ▼
[AIプロンプト: SpecDiffExtractor]  ※コードインタープリター ON
  - 両ファイルからテキスト/表を抽出
  - 差分(追加/削除/変更)を洗い出し
  - Pythonで openpyxl 等を使い xlsx を生成
        │ 出力: Artifact(xlsx), サマリテキスト
        ▼
[トピック: 後続処理]
  ④ Artifactをメッセージに添付して返却
  ⑤ サマリを本文で提示
```

## 2. トピック設計

### 2-1. 基本情報
| 項目 | 内容 |
|---|---|
| トピック名 | `仕様書変化点抽出` |
| トリガー方式 | 生成AIトリガーフレーズ（意味マッチ） |
| トリガーフレーズ例 | ・対象の仕様書から変化点を抽出して<br>・仕様書の差分を出して<br>・旧版と新版の変更点をまとめて<br>・スペックの変更箇所を教えて |

### 2-2. ノード構成（案）

1. **質問ノード（ファイル添付）**
   - 変数名: `Var_OldSpecFile`
   - プロンプト文言: 「比較元（旧版）の仕様書ファイルを添付してください」
   - 入力タイプ: ファイルの添付（.docx / .pdf / .xlsx 許可）

2. **質問ノード（ファイル添付）**
   - 変数名: `Var_NewSpecFile`
   - プロンプト文言: 「比較先（新版）の仕様書ファイルを添付してください」

3. **確認メッセージノード（任意）**
   - 「2ファイルを比較し、変化点をExcelにまとめます。少々お待ちください。」

4. **ツール呼び出しノード**
   - 呼び出し先: AIプロンプト `SpecDiffExtractor`
   - 入力マッピング:
     - `old_file` ← `Var_OldSpecFile`
     - `new_file` ← `Var_NewSpecFile`
   - 出力受け取り:
     - `Var_ResultArtifact`（生成ファイル）
     - `Var_ResultSummary`（テキスト要約）

5. **メッセージノード（ファイル返却）**
   - `Var_ResultArtifact` を添付して返信
   - 本文: 「変化点抽出結果をExcelにまとめました。以下からダウンロードできます。」

6. **メッセージノード（サマリ表示）**
   - `Var_ResultSummary` の内容を箇条書きで表示（主要な変更点トップNなど）

## 3. AIプロンプト（Prompt Builder）設計: `SpecDiffExtractor`

### 3-1. 入力/出力定義
| 種別 | 名前 | 型 |
|---|---|---|
| 入力 | old_file | ファイル |
| 入力 | new_file | ファイル |
| 出力 | diff_excel | ファイル（Artifact） |
| 出力 | summary_text | テキスト |

### 3-2. プロンプト指示文（たたき台）

```
あなたは仕様書比較の専門アシスタントです。
入力として旧版仕様書ファイル(old_file)と新版仕様書ファイル(new_file)を受け取ります。

手順:
1. コードインタープリターを使い、両ファイルの内容をテキスト/表として抽出する。
   - Word/PDF/Excelいずれの形式にも対応すること。
2. 章・節・項目単位で旧版と新版を突き合わせ、以下の観点で変化点を検出する。
   - 追加された項目
   - 削除された項目
   - 内容が変更された項目（変更前/変更後を明示）
3. 検出した変化点を以下の列を持つExcel(xlsx)として出力する。
   - No / 章番号 / 項目名 / 変更種別(追加・削除・変更) / 変更前内容 / 変更後内容 / 影響度(高/中/低) / 備考
   - 変更種別ごとに行の色分け（追加=緑、削除=赤、変更=黄）を行う。
4. 出力ファイル名は「変化点抽出結果_{現在日時}.xlsx」とする。
5. 主要な変化点（重要度が高いもの上位5件程度）をテキストで要約する。

出力は必ず diff_excel（Excelファイル）と summary_text（要約テキスト）の2つを返すこと。
比較対象の内容が読み取れない場合は、その旨をsummary_textに明記し、処理を中断すること。
```

### 3-3. コードインタープリターの役割
- ファイル形式変換・テキスト抽出（Word/PDF/Excel → 構造化テキスト）
- 差分アルゴリズム（章立てのマッチング、テキスト類似度による対応付け）の実行
- `openpyxl` 等によるxlsx生成（列幅調整・色分け・ヘッダー固定などの整形も含めてよい）
- 生成物をArtifactとして返却

## 4. 出力Excelのシート構成イメージ

| No | 章番号 | 項目名 | 変更種別 | 変更前内容 | 変更後内容 | 影響度 | 備考 |
|---|---|---|---|---|---|---|---|
| 1 | 3.2 | 認証方式 | 変更 | ID/PWのみ | ID/PW + MFA | 高 | セキュリティ要件強化 |
| 2 | 4.1 | 出力ファイル形式 | 追加 | （記載なし） | CSV出力オプション追加 | 中 | |

## 5. 代替構成案（コードインタープリターを使わない場合）

コードインタープリターがプレビュー機能で利用できない環境の場合、以下のPower Automateフロー構成でも実現可能。

1. トピックからPower Automateフロー（クラウドフロー）を呼び出し
2. フロー内でAI Builderの「プロンプトの実行」アクションにより変化点をJSON/テーブル形式で取得
3. 事前に用意したExcelテンプレート（テーブル定義済み）に対し、Excel Online (Business) コネクタの「テーブルに行を追加」を繰り返し実行（Apply to each）
4. 生成済みExcelファイルのOneDrive/SharePointリンクをトピックへ返却し、チャットに表示

この方式は列レイアウトが固定運用に向くが、コードインタープリター方式に比べて事前テンプレート管理の手間が増える。

## 6. 検証観点（PoC時のチェックリスト）

- [ ] 大きな仕様書（数十ページ）でもトークン制限内で処理できるか（分割要約が必要か）
- [ ] Word/PDF/Excel混在時の抽出精度
- [ ] 章番号がずれている場合（章立て変更）のマッチング精度
- [ ] 生成Excelのフォーマット崩れ（色分け・列幅）がないか
- [ ] トリガーフレーズの言い回しゆらぎへの追従（「差分」「変更点」「変化点」等）
- [ ] AI Builderクレジット消費量・実行時間

## 7. 今後の検討事項

- 比較対象ファイルを都度アップロードするのではなく、SharePoint上の最新版と前回版を自動参照する構成への発展
- 変化点の重要度判定ロジック（キーワードベース/LLM判定）の精緻化
- 複数仕様書（分冊）をまとめて比較するケースへの対応
